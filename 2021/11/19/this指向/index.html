JS中this指向

this指向

全局环境

在全局环境下，this 始终指向全局对象(window), 无论是否严格模式；

对于普通函数（包括匿名函数），this指的是直接的调用者，在非严格模式下，如果没有直接调用者，this指的是window。

函数直接调用：

普通函数内部的this分两种情况，严格模式和非严格模式。

(1)，非严格模式下，this 默认指向全局对象window

(2)，而严格模式下， this为undefined

四大规则

1.默认绑定规则

相当于函数的独立调用,默认指向window

2.隐式绑定规则

谁调用就指向谁

3.显示绑定

call,apply,bind用来绑定this指向

bind

多次 bind 时只认第一次 bind 的值

bind函数中this不会被修改

apply和call

apply() 和 call() 的第一个参数都是 this，区别在于通过 apply 调用时实参是放到数组中的，而通过 call 调用时实参是逗号分隔的。

父函数有能力决定子函数的this指向

注：高阶函数中的回调函数this指向为高阶函数api所决定的

可查阅forEach的MDN文档 如果不传值指向window

如果传入值 则指向传入的

4.new绑定

当使用 new 关键字调用函数时，函数中的 this 一定是 JS 创建的新对象。

如果构造函数返回原始值,比如return 1那么这个返回值毫无意义

例如下图 并不会改变test打印

如果构造函数返回值为对象，那么这个返回值会被正常使用

例如上面的 return {} 则打印为{}

使用new操作符调用函数，会自动执行以下步骤
  • 创建了一个全新的对象。
  • 这个对象会被执行[[Prototype]]（也就是__proto__）链接。
  • 生成的新对象会绑定到函数调用的this。
  • 通过new创建的每个对象将最终被[[Prototype]]链接到这个函数的prototype对象上。
  • 如果函数没有返回对象类型Object(包含Functoin, Array, Date, RegExg, Error)，那么new表达式中的函数调用会自动返回这个新的对象。
5.箭头函数

箭头函数没有this指针，箭头函数内的this指向外侧的this

箭头函数不能被当做构造函数

箭头函数的this不会被修改

new不能和箭头函数一起

6.不在函数里面

不在函数中的场景，可分为浏览器的 <script /> 标签里，或 Node.js 的模块文件里。

  1. 在 <script /> 标签里，this 指向 Window。
  2. 在 Node.js 的模块文件里，this 指向 Module 的默认导出对象，也就是 module.exports
7.setInterval和setTimeout等

都是属于window对象上的方法，那么this就是指向window了
